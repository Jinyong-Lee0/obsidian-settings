# Task Formats 소개

<span class="related-pages">#task-formats #index-pages</span>

## "작업 형식"이란 무엇인가요?

이 프로젝트에서 "작업 형식(Task Format)"이라는 용어는 작업 라인에서 작업의 속성인 마감일(Due date) 및 우선순위를 표현하는 데 사용되는 마크다운의 문자를 가리킵니다.

Tasks 플러그인이 생성된 이후, 작업 플러그인은 항상 이모지 문자(예: 📅 및 ⏫)만 사용했습니다.

## 작업 형식 선택

> [!릴리스]
> Tasks 3.3.0에서 소개되었습니다.

이제 Tasks가 작업 라인의 데이터를 읽고 쓰는 데 사용할 작업 형식을 선택할 수 있습니다:

![작업 형식을 선택하는 설정 옵션의 스크린샷](settings-task-format.png)

## 지원되는 작업 형식

Tasks는 다음 작업 형식을 지원합니다:

- [[Tasks 이모지 형식]] - 기본 형식
  - `⏫ 🔁 매일 완료될 때`
  - `🛫 2023-04-05 ⏳ 2023-04-06 📅 2023-04-07`
  - `➕ 2023-04-03 ✅ 2023-04-08`
- [[Dataview 형식]]
  - `[우선순위:: 높음], [반복:: 매일 완료될 때]`
  - `[시작일:: 2023-04-05], [예정일:: 2023-04-06], [마감일:: 2023-04-07]`
  - `[생성일:: 2023-04-03], [완료일:: 2023-04-08]`
  - **참고:** Tasks와 Dataview 간의 해석에 중요한 차이점이 있으므로 이 형식의 문서를 반드시 읽어보세요.

## 작업 형식이 Tasks 동작에 미치는 영향

> [!경고]
> 여러 형식에 대한 초기 지원에서 **Tasks는 한 번에 하나의 형식만 읽고 쓰는 것만 지원합니다**.
>
> Dataview 형식을 선택하면 Tasks는 자체 이모지 표시 기호를 더 이상 읽지 않습니다(다시 Tasks 형식으로 변경할 때까지).
>
> 이 사항은 [이슈 #1891](https://github.com/obsidian-tasks-group/obsidian-tasks/issues/1891)에서 추적되고 있으며, 앞으로 나올 릴리스에서 모든 지원되는 작업 형식을 *읽을 수 있도록* Tasks가 업데이트될 예정입니다.

### 선택한 형식에 따라 동작하는 Tasks 기능

선택한 작업 형식은 Tasks 플러그인이 작업 데이터를 읽고 쓰는 방식을 결정합니다.

다음 기능은 선택한 형식을 사용합니다:

- **작업 라인의 읽기 및 쓰기**
  - 이러한 기능은 **선택한 형식만** 사용합니다.
- **라이브 프리뷰**
- [[작업 만들기 또는 편집]] 모달은 선택한 형식으로 작업을 저장합니다.
  - 작업을 편집하는 동안 현재 값을 기본(이모지) 형식으로 표시하지만 참고하세요.
- [[자동 제안]]
  - Dataview 형식의 경우 먼저 선택한 대괄호 - `[]` 또는 `()` - 를 입력한 다음 대괄호 내에 삽입할 필드를 선택하여 입력해야 합니다.

### 선택한 형식을 고려하지 않는 Tasks 기능

여러 형식에 대한 초기 지원에서 다음 기능은 아직 선택한 형식을 사용하지 않으며 기본(이모지) 형식을 사용합니다:

- **읽기 모드**
  - 읽기 모드에서 작업 라인을 포함하는 노트를 보는 경우 사용자가 선택한 작업 형식과 상관없이 작업 데이터는 이모지로 표시됩니다.
  - 이 사항은 [이슈 #1889](https://github.com/obsidian-tasks-group/obsidian-tasks/issues/1889)에서 추적되고 있습니다.
- **작업 쿼리 블록에서 작업 표시**
  - 작업 검색 결과를 포함하는 노트를 보는 경우 선택한 작업 형식과 상관없이 작업 데이터는 이모지로 표시됩니다.
  - 이 사항은 [이슈 #1890](https://github.com/obsidian-tasks-group/obsidian-tasks/issues/1890)에서 추적되고 있습니다.

## 작업 형식 지원의 제한사항

- 작업 라인의 필드 순서는 여전히 중요합니다.
  - [[자동 제안#작업 내 항목의 순서에 대해 알아야 할 내용]]을 참조하세요.
  - 이 사항은 [이슈 #1505](https://github.com/obsidian-tasks-group/obsidian-tasks/issues/1505)에서 추적되고 있습니다.
- 현재 Tasks는 한 번에 하나의 형식만 지원합니다(위에서 언급한 대로).
  - Dataview 형식을 선택하면 Tasks는 이모지 표시 기호를 더 이상 읽지 않습니다.
  - 이 사항은 [이슈 #1891](https://github.com/obsidian-tasks-group/obsidian-tasks/issues/1891)에서 추적되고 있습니다.
- Tasks에서는 작업 형식을 다른 형식으로 전환하는 시설이 없습니다.
